Late 1970s-80s

VisiCalc

The first killer app

Huge productivity boost

VisiCalc was slow!

Replaced by Lotus 1-2-3

Reactivity

  • VisiCalc recalculated "everything" on every change
  • Lotus 1-2-3 only recalculated based on changed cells
  • Reactivity is "reacting to changes"
  • Reactivity = Performance

The unfortunate truth

  • The popular frameworks web aren't (fully) reactive!
  • React is not reactive!
  • Angular is not reactive!

What is velte

velte is

  • A Reactive web framework
  • Simple to use
  • Fast!

Simple, how?

  • Simple reactive constructs
  • Compiler does the hard stuff
  • support built-in

Fast, how?

  • Build using ite
  • ite is faster due to prebundling dependencies, and separating source code from dependencies.
  • Simpler code is faster to write
  • Less thought about reactivity = faster to write

How-to use velte

Structure

  • One Component per file
  • Components are named uppercase
  • .svelte extension

Component properties

React

Angular

Component properties

  • export exposes properties
  • export is normally used for exposing constructs in ESM
  • export is used by the compiler, differently!

Component events

React

Angular

Component events

  • Use createEventDispatcher
  • Propogate the event to the parent, using the dispatcher
  • Dispatched event can have payload

Reactivity

React

Angular

Reactivity

  • $: is a JavaScript label
  • In normal JavaScript, this can be break'd or continue'd to
  • In velte, this is a compiler hint!

Templates blocks

velte supports templated logic constructs:

  • if blocks
  • each blocks (loops)
  • await blocks (promise handling)

If blocks

React

Angular

If blocks

  • Similar to imperative if statements
  • Notice different usages of #, : and /

Loops

React

Angular

Loops (each blocks)

  • Similar to imperative for-each statements
  • Nice handling of "empty iterables" with :else-block

Promises

React

Angular

Promises (await blocks)

  • Similar to imperative .then(...).catch(...) usage
  • Optional pre-resolve state (move then to await block)
  • catch block can be omitted
